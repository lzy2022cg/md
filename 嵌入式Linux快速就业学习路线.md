# 嵌入式 Linux 快速就业学习路线

> 基于正点原子 i.MX6ULL 三本教材制定，目标：**最短时间内具备初级嵌入式 Linux 岗位的核心技能**
> 三本教材：
> - 《I.MX6U 嵌入式 Linux C 应用编程指南 V1.6》（简称：**C应用**）
> - 《I.MX6U 嵌入式 Linux 驱动开发指南 V2.0.1》（简称：**驱动**）
> - 《I.MX6U 嵌入式 Qt 开发指南 V1.2》（简称：**Qt**）

---

## 岗位方向选择（先确定，再学习）

| 岗位 | 主攻方向 | 预计学习周期 |
|------|---------|------------|
| 嵌入式 Linux 应用开发 | C应用 全部 + 驱动少量 | **6~8 周** |
| 嵌入式 Linux 驱动/BSP | C应用 核心 + 驱动全部 | **10~14 周** |
| 嵌入式 Qt 界面开发 | C应用 核心 + Qt 全部 | **8~10 周** |

---

## 第一阶段：地基（约 1 周）

> 无论哪个方向，必须先过一遍。

### 来源：《驱动》第1~4章 + 《C应用》第1章

| 章节 | 内容 | 重要程度 |
|------|------|---------|
| 驱动 第1章 | Ubuntu 系统安装与基本使用 | ★★★ |
| 驱动 第2章 | Ubuntu 系统入门（Shell、文件系统、权限） | ★★★ |
| 驱动 第3章 | Linux C 编程入门（gcc、Makefile） | ★★★★ |
| 驱动 第4章 | 开发环境搭建（VSCode、交叉编译器、SecureCRT/MobaXterm） | ★★★★ |
| C应用 第1章 | 应用编程概念（系统调用 vs 库函数） | ★★★ |

**目标：** 能在 Ubuntu 下写 C 程序并用 Makefile 编译，能通过串口连接开发板。

---

## 第二阶段：Linux C 应用开发（约 3~4 周）⭐ 核心

> 这是找工作最直接有用的部分，面试高频考点集中在此。

### 来源：《C应用》第2~13章

#### 第一轮：文件 I/O（必须精通）

| 章节 | 内容 | 重点知识点 |
|------|------|----------|
| C应用 第2章 | 文件 I/O 基础 | open / read / write / close / lseek、文件描述符 |
| C应用 第3章 | 深入文件 I/O | inode、O_APPEND、原子操作、fcntl、ioctl |
| C应用 第4章 | 标准 I/O 库 | FILE 指针、fopen/fread/fwrite、格式化 I/O、I/O 缓冲机制 |
| C应用 第5章 | 文件属性与目录 | stat / fstat、文件权限、软硬链接、目录操作 |

#### 第二轮：进程与线程（面试必考）

| 章节 | 内容 | 重点知识点 |
|------|------|----------|
| C应用 第8章 | 信号 | signal / sigaction、信号集、信号掩码、kill / raise |
| C应用 第9章 | 进程 | fork / exec / wait、守护进程、僵尸进程、孤儿进程 |
| C应用 第10章 | 进程间通信概述 | 管道、FIFO、消息队列、共享内存、信号量（了解概念即可） |
| C应用 第11章 | 线程 | pthread_create / join / detach、线程取消、线程属性 |
| C应用 第12章 | 线程同步 | **互斥锁、条件变量、自旋锁、读写锁**（面试最高频） |

#### 第三轮：高级 I/O（重要）

| 章节 | 内容 | 重点知识点 |
|------|------|----------|
| C应用 第13章 | 高级 I/O | 非阻塞 I/O、select / poll 多路复用、mmap 存储映射、文件锁 |

#### 可选（提高竞争力）

| 章节 | 内容 | 说明 |
|------|------|------|
| C应用 第6章 | 字符串处理 | 查漏补缺 |
| C应用 第7章 | 系统信息与资源 | 时间函数、malloc/free 细节、proc 文件系统 |

**阶段目标：** 写出多线程服务端程序，用互斥锁保护共享资源，用 select 实现 I/O 多路复用。

## 第三阶段：Linux 应用层硬件操作（约 1 周）

> 《C应用》第二篇"提高篇"，学习如何从应用层操控硬件。

### 来源：《C应用》第15~28章（选学）

| 章节 | 内容 | 建议 |
|------|------|------|
| C应用 第15章 | 点亮 LED（sysfs 文件系统） | **必学**，理解 Linux 硬件控制两种方式 |
| C应用 第16章 | GPIO 应用编程 | **必学**，输入/输出/中断 |
| C应用 第17章 | 输入设备应用编程 | **必学**，input 子系统、触摸屏、按键 |
| C应用 第19章 | FrameBuffer 应用编程 | 必学，LCD 控制、BMP 图片显示 |
| C应用 第25章 | V4L2 摄像头应用编程 | 选学，做项目时再看 |
| C应用 第26章 | 串口应用编程 | 必学，termios 结构体、串口配置 |
| C应用 第29~30章 | 网络基础 + socket 编程 | **必学**，TCP/UDP 编程是面试重点 |
| C应用 第34章 | 实战：MQTT 物联网 / 视频监控项目 | 选学，简历项目用 |

## 第四阶段：Linux 驱动开发核心（约 3~4 周）⭐ 驱动岗必学

> 不需要学完全部驱动章节，以下是最具性价比的核心章节。

### 来源：《驱动》第40~55章

#### 第一步：字符设备驱动三件套

| 章节 | 内容 | 核心知识点 |
|------|------|----------|
| 驱动 第40章 | 字符设备驱动开发 | `register_chrdev`、`file_operations`、驱动模块加载/卸载 |
| 驱动 第41章 | 嵌入式 Linux LED 驱动 | `ioremap` 映射寄存器、完整驱动 + 测试 APP 流程 |
| 驱动 第42章 | 新字符设备驱动 | `alloc_chrdev_region`、`cdev`、自动创建设备节点（mdev） |

#### 第二步：设备树（现代驱动开发必须）

| 章节 | 内容 | 核心知识点 |
|------|------|----------|
| 驱动 第43章 | Linux 设备树 | DTS 语法、节点属性、OF 操作函数族 |
| 驱动 第44章 | 设备树下的 LED 驱动 | 从设备树获取 GPIO 资源，完整实战 |
| 驱动 第45章 | pinctrl 和 gpio 子系统 | `pinctrl` 配置、`gpio_request / gpio_direction_output` |

#### 第三步：驱动并发与中断（面试考点）

| 章节 | 内容 | 核心知识点 |
|------|------|----------|
| 驱动 第47章 | Linux 并发与竞争（理论） | 原子操作、自旋锁、信号量、互斥体 |
| 驱动 第48章 | Linux 并发与竞争实验 | 实际代码验证 |
| 驱动 第50章 | Linux 内核定时器实验 | `timer_list`、内核时间管理 |
| 驱动 第51章 | Linux 中断实验 | `request_irq`、中断处理函数、`tasklet` |

#### 第四步：platform 框架（企业驱动开发标准范式）

| 章节 | 内容 | 核心知识点 |
|------|------|----------|
| 驱动 第52章 | Linux 阻塞和非阻塞 IO | `wait_queue`、poll 机制 |
| 驱动 第53章 | 异步通知实验 | `fasync`、`kill_fasync` |
| 驱动 第54章 | platform 设备驱动实验 | `platform_driver`、`platform_device`、`probe` 函数 |
| 驱动 第55章 | 设备树下的 platform 驱动 | 现代驱动开发完整范式，**最终目标** |

**阶段目标：** 能独立编写基于设备树 + platform 框架的字符设备驱动，含中断和并发处理。

## 总时间规划

| 阶段 | 内容 | 建议时间 | 优先级 |

| 第一阶段 | Linux 地基 + 开发环境 | 1 周 | 必须 |

## 简历关键词参考

学完以上内容后，简历可以写：

**Linux C 应用开发方向：**
- 熟练掌握 Linux 文件 I/O（系统调用与标准库）
- 掌握多进程编程（fork/exec/wait）及进程间通信（管道/消息队列/共享内存）
- 熟练掌握多线程编程（pthread）及线程同步（互斥锁/条件变量/读写锁）
- 掌握 I/O 多路复用（select/poll）及 mmap 存储映射
- 掌握 Linux Socket 网络编程（TCP/UDP）
- 了解 Linux 应用层硬件操控（GPIO/输入设备/FrameBuffer/串口）

**Linux 驱动开发方向（在上面基础上加）：**
- 掌握 Linux 字符设备驱动开发（chrdev/cdev/file_operations）
- 熟悉 Linux 设备树（DTS 语法、OF 操作函数）
- 掌握 pinctrl 和 GPIO 子系统驱动开发
- 掌握 Linux 内核并发机制（原子操作/自旋锁/信号量/互斥体）
- 掌握 Linux 中断子系统（request_irq、tasklet）
- 掌握 platform 驱动框架开发
