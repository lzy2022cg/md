# 嵌入式 Linux 应用开发 — 快速就业学习路线

> 目标岗位：**嵌入式 Linux 应用开发工程师**
> 参考教材：《I.MX6U 嵌入式 Linux C 应用编程指南 V1.6》（主线）
>           《I.MX6U 嵌入式 Linux 驱动开发指南 V2.0.1》（辅助，仅用前4章）
> 预计周期：**6~8 周**（每天 4~6 小时）

---

## 岗位说明

嵌入式 Linux 应用开发工程师的工作内容：

- 在已有 Linux 系统上编写运行于用户空间的 C/C++ 程序
- 通过 `/dev` 设备文件、`sysfs`、`ioctl` 等接口操控硬件
- 编写多进程 / 多线程服务程序
- 编写网络通信程序（TCP/UDP/MQTT）
- 调用 V4L2、FrameBuffer、串口等 Linux 标准接口

**不需要写驱动，不需要移植内核，不需要改 U-Boot。**

## 第二阶段：文件 I/O 体系（第 2~2.5 周）

> 来源：《C应用》第 1~5 章
> **文件 I/O 是 Linux 应用开发的根基，一切设备操作本质上都是文件操作。**

### 第1章 应用编程概念

| 知识点 | 说明 |
|-------|------|
| 系统调用 vs 库函数 | 两者区别，`strace` 工具追踪系统调用 |
| 标准 C 语言函数库 | 了解 glibc 的作用 |

### 第2章 文件 I/O 基础（必须精通）

| 知识点 | 关键函数 |
|-------|---------|
| 打开 / 关闭文件 | `open()` / `close()`，flags 参数：`O_RDONLY` `O_WRONLY` `O_RDWR` `O_CREAT` |
| 读写文件 | `read()` / `write()`，返回值含义 |
| 文件偏移 | `lseek()`，`SEEK_SET` `SEEK_CUR` `SEEK_END` |
| 文件描述符 | 0/1/2 标准 IO，最大打开数限制 |

### 第3章 深入文件 I/O（重要）

| 知识点 | 说明 |
|-------|------|
| inode 与文件 | 静态文件与打开文件的区别 |
| 错误处理 | `errno` / `perror()` / `strerror()` |
| 原子操作 | `O_APPEND` 标志，竞争条件问题 |
| 文件复制 | `dup()` / `dup2()`，重定向原理 |
| fcntl / ioctl | 文件控制，设备驱动通信的核心接口 |

### 第4章 标准 I/O 库

| 知识点 | 关键函数 |
|-------|---------|
| FILE 指针 | `fopen()` / `fclose()` |
| 读写操作 | `fread()` / `fwrite()` / `fgets()` / `fputs()` |
| 格式化 I/O | `fprintf()` / `fscanf()` / `sscanf()` |
| I/O 缓冲 | 全缓冲 / 行缓冲 / 无缓冲，`fflush()` |
| 文件描述符互转 | `fileno()` / `fdopen()` |

### 第5章 文件属性与目录

| 知识点 | 关键函数 |
|-------|---------|
| 获取文件信息 | `stat()` / `fstat()` / `lstat()`，`struct stat` 结构体 |
| 文件类型判断 | `S_ISREG` `S_ISDIR` `S_ISCHR` 宏 |
| 文件权限 | `chmod()` / `access()` / `umask()` |
| 软链接与硬链接 | `symlink()` / `readlink()` |
| 目录操作 | `opendir()` / `readdir()` / `closedir()` / `mkdir()` |

### 阶段目标

- 能写一个文件复制程序（支持大文件、错误处理）
- 理解 `open` 和 `fopen` 的区别及使用场景
- 知道 `ioctl` 是应用层与驱动通信的核心手段

## 第三阶段：进程 / 线程 / 同步（第 3~4.5 周）⭐ 面试最高频

> 来源：《C应用》第 7~13 章
> **这部分是面试必考内容，每个知识点都要能手写代码。**

### 第7章 系统信息与系统资源

| 知识点 | 关键函数 |
|-------|---------|
| 时间操作 | `time()` / `gettimeofday()` / `localtime()` / `strftime()` |
| 内存申请 | `malloc()` / `calloc()` / `realloc()` / `free()`，内存泄漏概念 |
| proc 文件系统 | `/proc/cpuinfo` `/proc/meminfo` `/proc/PID/` 目录结构 |

### 第8章 信号（重要）

| 知识点 | 说明 |
|-------|------|
| 信号概念 | 信号是异步通知机制，类比硬件中断 |
| 常用信号 | `SIGINT` `SIGTERM` `SIGKILL` `SIGCHLD` `SIGUSR1/2` |
| 信号处理 | `signal()` / `sigaction()`（推荐用后者） |
| 向进程发送信号 | `kill()` / `raise()` / `alarm()` |
| 信号集操作 | `sigemptyset` / `sigaddset` / `sigprocmask` |

### 第9章 进程（重要）

| 知识点 | 关键函数 | 面试考点 |
|-------|---------|---------|
| 进程概念 | PID、虚拟地址空间、进程状态 | 进程与线程区别 |
| 创建子进程 | `fork()`，父子进程的资源继承规则 | fork 后各变量的值 |
| 执行新程序 | `execve()` / `exec` 族函数 | exec 不创建新进程 |
| 等待子进程 | `wait()` / `waitpid()`，`WIFEXITED` / `WEXITSTATUS` 宏 | 避免僵尸进程 |
| 僵尸进程 | 子进程退出父进程未 wait | 如何避免 |
| 孤儿进程 | 父进程先退出，子进程被 init 收养 | 概念理解 |
| 守护进程 | `setsid()` / `fork()` 两次 / 关闭文件描述符 | 写法要会 |

### 第10章 进程间通信（了解为主）

| IPC 方式 | 特点 | 适用场景 |
|---------|------|---------|
| 无名管道 `pipe()` | 只能父子进程间使用，半双工 | 父子进程通信 |
| 有名管道 `mkfifo()` | 任意进程间，文件系统中有名字 | 无亲缘关系进程 |
| 信号量 | 控制并发访问共享资源 | 进程同步 |
| 共享内存 | 最快的 IPC 方式 | 大数据量传输 |
| 消息队列 | 有类型的消息，异步通信 | 解耦生产者消费者 |
| Socket | 可跨网络，最通用 | 网络通信（重点单独学） |

### 第11章 线程（必须精通）

| 知识点 | 关键函数 |
|-------|---------|
| 创建线程 | `pthread_create()`，函数原型、参数含义 |
| 等待线程结束 | `pthread_join()`，获取返回值 |
| 线程分离 | `pthread_detach()`，分离后不能 join |
| 取消线程 | `pthread_cancel()`，取消点概念 |
| 线程退出 | `pthread_exit()` vs `return` vs `exit()` 的区别 |
| 线程属性 | `pthread_attr_t`，设置栈大小、分离状态 |
| 线程局部存储 | `pthread_key_create()`，每个线程独立的变量 |

### 第12章 线程同步（必须精通，面试必考）

| 同步机制 | 关键函数 | 使用场景 |
|---------|---------|---------|
| **互斥锁** | `pthread_mutex_lock/unlock/trylock` | 保护临界区，最常用 |
| **条件变量** | `pthread_cond_wait/signal/broadcast` | 线程间等待某个条件成立 |
| 自旋锁 | `pthread_spin_lock/unlock` | 临界区极短、不能睡眠的场景 |
| 读写锁 | `pthread_rwlock_rdlock/wrlock/unlock` | 读多写少的共享数据 |
| 死锁 | 产生条件、如何避免 | 面试必问 |

> **手写要求：** 生产者-消费者模型（互斥锁 + 条件变量实现），这是面试最常考的编程题。

### 第13章 高级 I/O（重要）

| 知识点 | 关键函数 | 说明 |
|-------|---------|------|
| 非阻塞 I/O | `O_NONBLOCK` 标志 | read/write 不阻塞，立即返回 |
| **I/O 多路复用** | `select()` / `poll()` | **同时监听多个文件描述符，面试必考** |
| 异步 I/O | `SIGIO` 信号 / `aio_*` | 了解概念 |
| **存储映射 I/O** | `mmap()` / `munmap()` | LCD 显示、大文件操作的核心手段 |
| 文件锁 | `flock()` / `fcntl()` 加锁 | 多进程访问同一文件时的保护 |

### 阶段目标

- 能手写生产者-消费者模型（多线程 + 互斥锁 + 条件变量）
- 能写出 select 监听多个文件描述符的程序
- 能解释进程与线程的区别、僵尸进程的产生与避免
- 理解 mmap 的原理，能用 mmap 操作 LCD 显示

## 第四阶段：网络编程（第 5 周）⭐ 就业必备

> 来源：《C应用》第 29~30 章

### 第29章 网络基础知识

| 知识点 | 说明 |
|-------|------|
| OSI 七层模型 / TCP/IP 四层模型 | 了解各层作用 |
| TCP vs UDP | 有连接 vs 无连接、可靠 vs 不可靠、应用场景区别 |
| IP 地址 / 端口号 | 网络字节序 `htons()` / `ntohl()` |
| 套接字类型 | `SOCK_STREAM`（TCP）/ `SOCK_DGRAM`（UDP） |

### 第30章 Socket 编程基础（必须手写）

#### TCP 编程（服务端 + 客户端）

```
服务端流程：                    客户端流程：
socket()                        socket()
bind()                              ↓
listen()                        connect()
accept()  ←── 连接 ───→         send() / recv()
recv() / send()                 close()
close()
```

| 关键函数 | 说明 |
|---------|------|
| `socket()` | 创建套接字 |
| `bind()` | 绑定地址和端口 |
| `listen()` | 监听连接请求 |
| `accept()` | 接受客户端连接，返回新的 fd |
| `connect()` | 客户端发起连接 |
| `send()` / `recv()` | 发送 / 接收数据 |
| `setsockopt()` | 设置 `SO_REUSEADDR` 等 socket 选项 |

#### UDP 编程

| 关键函数 | 说明 |
|---------|------|
| `sendto()` / `recvfrom()` | UDP 无连接收发，需指定目标地址 |
| 广播 | `SO_BROADCAST` socket 选项 |

#### 多客户端处理方案（面试考点）

| 方案 | 实现方式 | 优缺点 |
|-----|---------|-------|
| 多进程 | `fork()` 每个连接 | 隔离好，开销大 |
| 多线程 | `pthread_create()` 每个连接 | 轻量，需注意线程安全 |
| I/O 多路复用 | `select()` / `poll()` | 单线程处理多连接，高效 |

### 阶段目标

- 能手写 TCP 回声服务器（echo server）
- 能实现基于 select 的多客户端并发服务器
- 理解 TCP 三次握手 / 四次挥手（面试必问）

### 第16章 GPIO 应用编程（必学）

|-------|------|

### 第17章 输入设备应用编程（必学）

### 第19章 FrameBuffer 应用编程（必学）

### 第24章 PWM 应用编程（选学）

|-------|------|

### 第26章 串口应用编程（必学）

| 知识点 | 关键函数 |
|-------|---------|
| 打开串口设备 | `open("/dev/ttyS0", ...)` |

| 设置波特率 | `cfsetispeed()` / `cfsetospeed()` |

### 第25章 V4L2 摄像头（选学，项目时用）

### 阶段目标

- 能读取触摸屏 / 按键事件

## 第六阶段：实战项目（第 7~8 周）

> 来源：《C应用》第 31~34 章（CMake + MQTT + 视频监控）

### 推荐项目方向（二选一）

#### 方案 A：网络数据采集系统
- 多线程采集串口传感器数据（温湿度、光照等）

- PC 端用 select 实现多客户端并发接收并打印

**用到的技能：** 串口、多线程、互斥锁、TCP Socket、select

#### 方案 B：MQTT 物联网小项目（第34章）
- 在开发板上运行 MQTT 客户端
- 采集 GPIO / 传感器数据，通过 MQTT 发布到云端
- 订阅控制指令，控制 LED / 蜂鸣器

**用到的技能：** 网络编程、多线程、GPIO 控制、MQTT 协议

### CMake 工程管理（第32~33章）

- 替代复杂 Makefile 的现代构建工具
